# 二、技术路线

## 1. 项目选型

- 语言：`C++17`
- 构建：`CMake`
- 测试：`GoogleTest`
- 解码：`FFmpeg`

# 三、你当前最需要补的知识

这部分不是“了解概念”，而是必须达到“能写代码 + 能解释为什么”。

## 1. 并发基础

必须掌握：

- `std::thread`
- `std::mutex`
- `std::unique_lock`
- `std::condition_variable`
- `std::atomic`
- 生产者消费者模型
- 线程退出和资源回收

最低面试表达要求：

- 能解释为什么要用 `condition_variable`
- 能解释什么是虚假唤醒
- 能说明 `close()` / `stop flag` 为什么重要
- 能说清楚为什么线程退出顺序不对会死锁

## 2. CMake 基础

必须掌握：

- `add_library`
- `add_executable`
- `target_link_libraries`
- `target_include_directories`
- `PUBLIC / PRIVATE / INTERFACE`
- `FetchContent`
- `enable_testing`
- `add_test` / `ctest`

最低面试表达要求：

- 能解释 target-based CMake 的基本思路
- 能说明为什么不要全局乱加 include/lib
- 能自己把一个库、一个可执行程序、一个测试 target 串起来

## 3. GoogleTest 基础

必须掌握：

- `TEST`
- `EXPECT_EQ / EXPECT_TRUE`
- `ASSERT_*`
- fixture 基本概念
- 如何用 `ctest` 跑测试

最低面试表达要求：

- 能解释为什么并发代码更需要单测
- 能说明单测和手工跑 demo 的区别

# 八、风险提醒

这 12 周里最容易失败的不是“学不会”，而是下面 4 个问题：

1. **一开始就把项目做太重**
